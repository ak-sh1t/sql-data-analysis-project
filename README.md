## Telecom Customer Churn Analysis — SQL Project

An end-to-end SQL data analysis project identifying churn patterns,
revenue impact, and high-risk customers in a telecom business using MySQL.


## Project Overview

Customer churn directly impacts revenue and growth. This project analyzes
7,000+ telecom customer records using MySQL to uncover why customers leave,
which segments churn most, and which customers need immediate retention focus.


## Business Questions Answered

- What is the overall churn rate?
- Which contract types and payment methods drive churn?
- Does internet service type affect churn?
- How do charges differ between churned vs retained customers?
- Which customers are high-risk and need retention action?
- How much revenue comes from each contract group?


## Dataset
- Source: [IBM Telco Customer Churn — Kaggle](https://www.kaggle.com/datasets/blastchar/telco-customer-churn)
- Records: 7,043 customers
- Format: CSV


## Tools Used

 - MySQL for Data cleaning, analysis, views 
 - MySQL Workbench for Query execution & schema management 
 - GitHub for Version control & documentation 


## Key Findings

- Overall churn rate is ~26%
- Month-to-month contracts have the highest churn (~42%)
- Fiber Optic users churn more than DSL users
- Electronic check payment method shows the highest churn rate
- New customers (< 12 months tenure) are most likely to churn
- Churned customers pay higher average monthly charges


## Business Recommendations

1. Retention offers for High Risk customers before month 12
2. Incentivize annual/2-year contracts with pricing discounts
3. Investigate Fiber Optic pricing — high charges correlate with churn
4. Simplify Electronic Check process — possible payment friction
